# 無人販売支援システム 管理画面・通知機能

## 概要

このプログラムは、無人販売所における商品管理、在庫管理、売上管理、通知管理を行うためのWeb管理画面です。

管理者はWeb画面上から、商品の登録・更新、現在の在庫確認、売上履歴の確認、通知履歴の確認、AI認識結果履歴の確認を行うことができます。

また、購入時や万引き疑い発生時には、LINE通知を送信する機能も実装しています。

## 各ファイルの詳細

### app.py

`app.py` は、Flaskアプリケーション本体のファイルです。

このファイルでは、管理画面の表示と、管理画面から利用する各種APIの処理を行っています。  
商品情報、在庫情報、売上履歴、通知履歴、AI認識結果履歴を取得するAPIを定義し、Web画面から情報を確認できるようにしています。

また、管理画面から商品を登録・更新したり、販売しなくなった商品を削除したりする処理も `app.py` で行っています。

購入処理では、購入された商品の在庫数を減らし、売上履歴を追加し、LINEで購入通知を送信します。  
万引き通知処理では、商品名、個数、不足金額をもとに通知履歴を追加し、LINEで万引き通知を送信します。

AI認識結果を受け取るAPIもあり、AIから送られた商品名と個数をもとに在庫数を更新します。  
ただし、削除済みの商品や未登録の商品については、AI認識結果に含まれていても再登録せず、無視するようにしています。

主な役割は以下です。

- 管理画面の表示
- 商品情報の取得
- 在庫情報の取得
- 売上履歴の取得
- 通知履歴の取得
- AI認識履歴の取得
- 商品登録・更新
- 商品削除
- 購入処理
- 万引き通知処理
- AI認識結果による在庫更新
- LINE通知処理との連携

### data_store.py

`data_store.py` は、商品情報、在庫情報、売上履歴、通知履歴、AI認識履歴を管理するファイルです。

このファイルでは、商品名や価格を管理する `products`、現在の在庫数を管理する `inventory`、売上履歴を保存する `sales_history`、通知履歴を保存する `notification_history`、AI認識結果の履歴を保存する `ai_history` を用意しています。

商品登録・更新では、商品名、価格、在庫数をまとめて登録または更新します。  
販売しなくなった商品を削除する場合は、商品情報と在庫情報の両方から対象の商品を削除します。

在庫更新には2種類あります。  
1つ目は、購入時などに在庫数を増減する処理です。  
2つ目は、AI認識結果をもとに在庫数を上書きする処理です。

AI認識結果による在庫更新では、未登録の商品や削除済みの商品を再追加しないようにしています。

また、売上履歴、通知履歴、AI認識履歴には、記録時刻も一緒に保存されます。  
これにより、いつ購入が発生したのか、いつ通知が送られたのか、いつAI認識結果が反映されたのかを後から確認できます。

主な役割は以下です。

- 商品情報の管理
- 在庫情報の管理
- 売上履歴の管理
- 通知履歴の管理
- AI認識履歴の管理
- 商品登録・更新
- 商品削除
- 購入時の在庫数更新
- AI認識結果による在庫数更新
- 売上履歴の追加
- 通知履歴の追加
- AI認識履歴の追加

### line_notify.py

`line_notify.py` は、LINE通知を送信するためのファイルです。

このファイルでは、LINE Messaging APIを用いて、購入通知や万引き通知を送信します。  
LINE通知には、環境変数 `LINE_CHANNEL_ACCESS_TOKEN` に設定したチャネルアクセストークンを使用します。

購入通知では、商品名、個数、支払金額をLINEに送信します。  
万引き通知では、商品名、個数、不足金額をLINEに送信します。

また、チャネルアクセストークンが設定されていない場合でも、プログラムが停止しないようになっています。  
その場合は、LINEには通知を送らず、通知内容をコンソールに表示します。

これにより、LINE通知をまだ設定していない開発段階でも、通知内容を確認しながら動作テストができます。

主な役割は以下です。

- LINE Messaging APIによる通知送信
- 環境変数からLINEアクセストークンを取得
- 購入通知の送信
- 万引き通知の送信
- LINE通知未設定時のコンソール出力
- 通知送信結果の確認

### templates/index.html

`templates/index.html` は、管理画面のHTMLファイルです。

このファイルでは、管理者が操作するWeb画面の構成を作成しています。  
画面には、商品登録・更新、現在の在庫、売上履歴、通知履歴、AI認識結果履歴、動作テストの項目があります。

商品登録・更新では、商品名、価格、在庫数を入力し、登録・更新ボタンを押すことで商品情報を変更できます。  
現在の在庫では、商品名、価格、在庫数を一覧で表示し、販売しなくなった商品は削除ボタンで削除できます。

売上履歴では、購入時刻、商品名、個数、金額を表示します。  
通知履歴では、購入通知や万引き通知の内容を一覧で確認できます。  
AI認識結果履歴では、AIから送信された商品名と認識個数を確認できます。

また、このファイルにはJavaScriptも含まれており、Flask側のAPIと通信して画面を更新します。  
例えば、商品を登録・更新した後には在庫一覧を更新し、購入通知テストを行った後には在庫、売上履歴、通知履歴を更新します。

さらに、5秒ごとに在庫や履歴を自動更新する処理も入っています。

主な役割は以下です。

- 管理画面の表示
- 商品登録・更新フォームの表示
- 現在の在庫一覧の表示
- 売上履歴の表示
- 通知履歴の表示
- AI認識結果履歴の表示
- 動作テストボタンの表示
- Flask APIとの通信
- 画面情報の自動更新

### static/style.css

`static/style.css` は、管理画面の見た目を整えるCSSファイルです。

このファイルでは、画面全体の背景色、文字、ヘッダー、カード形式のレイアウト、表、ボタン、入力フォームなどのデザインを設定しています。

管理画面では、各機能をカード形式で分けて表示しています。  
これにより、商品登録・更新、在庫表示、売上履歴、通知履歴、AI認識結果履歴、動作テストが見やすく整理されています。

ボタンには、通常の操作ボタン、削除や万引き通知に使う危険操作用の赤いボタン、AI認識結果テスト用の青いボタンを設定しています。  
これにより、操作の種類が視覚的に分かりやすくなっています。

また、入力フォームは横並びで表示され、商品名、価格、在庫数を入力しやすいようにしています。  
表は罫線や背景色を設定し、商品情報や履歴を見やすくしています。

主な役割は以下です。

- 管理画面全体のデザイン設定
- ヘッダーのデザイン設定
- カード形式レイアウトの設定
- 表のデザイン設定
- ボタンの色や形の設定
- 削除ボタンの強調表示
- AI認識ボタンの強調表示
- 入力フォームのデザイン設定
- 画面全体の見やすさの向上
