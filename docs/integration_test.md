# 結合テスト手順書

結合テストの手順・成功条件・トラブル対策のまとめ。
mainにマージされた全モジュール（AI認識・制御・万引き判定・Web管理画面）を前提とする。

---

## 1. システム全体像と各担当モジュール

```
[人検知] → セッション開始 → [録画] ─┐
     ↓                              │
[コイン認識] (0.2秒周期) → coin.csv  ├→ セッション終了時
     ↓                              │   ├ vegetable.csv / weight.csv 保存
[野菜認識] → vegetable.csv          ┘   ├ theft_checker.py 起動
[重量センサー] → weight.csv              └ session.json 生成
                                              ↓
                              Flask管理画面が sessions/ を監視して自動取込
                              → 在庫更新・売上履歴・LINE通知
```

| モジュール | ファイル | 状態 |
|-----------|---------|------|
| AI推論サーバー | `app/web_server.py` + `ai/inference.py` | ✅ 単体動作確認済み |
| セッション制御 | `app/controller.py` | ⚠️ AI部分が**ダミー実装**（後述） |
| 万引き判定 | `app/theft_checker.py` | CSV入力で動作 |
| Web管理画面・LINE通知 | `Flask/app.py` | sessions/ 監視で自動取込 |
| ラズパイクライアント | `app/main.py`, `app/simulate_raspi.py` | ✅ 実装済み（実機未検証） |

### 最重要: AI と制御の結合ポイント

`app/controller.py` には AI担当の実装への**差し替えを待つダミー関数が3つ**ある：

| ダミー関数 | 現在の動作 | 差し替え後の動作（AI側の責務） |
|-----------|-----------|------------------------------|
| `detect_person()` → `bool` | キーボードで y/n 入力 | `/predict` の結果に `person` が含まれるか |
| `detect_coin()` → `list[int]` | キーボードで金額入力 | `/predict` の `100yen` 等の検出を `[100, ...]` に変換 |
| `detect_vegetables()` → `dict` | 固定値を返す | `/predict` の野菜検出を `{"eggplant": 4}` 形式に集計 |

**結合テストの核心は、このダミー3関数をAI実装に差し替えて全体フローが回ることの確認。**

### カメラ構成（`app/config.py`）— 2台構成

| カメラ | index | 用途 |
|--------|-------|------|
| 監視カメラ | 0 | 人検知・録画 |
| コイン・野菜カメラ | 1 | 硬貨認識と野菜認識で共用 |

※ 同じindexを指定すると内部で1つのカメラを自動共有する（設定変更だけで台数を増減可能）。
※ `scripts/start-container.sh` は /dev/video0〜2 を自動でコンテナに渡す（存在チェック付き）。

### 実行場所の前提（実機構成）

| プロセス | 実行場所 | 理由 |
|---------|---------|------|
| `app/web_server.py`（AI推論） | **Ubuntu GPUサーバー** | GPU必須 |
| `app/controller.py`（セッション制御） | **ラズパイ** | カメラ・GPIO・重量センサーに接続 |
| `app/raspberry_pi.py`（重量） | ラズパイ（controllerから呼ばれる） | HX711はGPIO直結 |
| `Flask/app.py`（管理画面・LINE） | ラズパイ（sessions/ を直接監視するため） | 設計上のパスがラズパイ前提 |

## ⚠️ 実機テスト前に必ず対処が必要な確定問題

コードレビューで判明した、**現状のまま実機で動かすと確実に失敗する問題**。

### 問題1: ~~起動場所によって theft_checker が起動できない~~ → ✅ 修正済み

launcher.py が theft_checker.py を絶対パスで起動するよう修正済み。
**どのディレクトリから controller.py を起動してもクラッシュしなくなった**
（動作検証済み）。

### 問題2: ~~controllerとFlaskでセッションフォルダの場所が違う~~ → ✅ 修正済み

Flask の監視先デフォルトを controller の保存先（`<リポジトリルート>/sessions/`）に
統一済み。環境変数なしでそのまま起動してよい。
別の場所を監視したい場合のみ `SESSIONS_DIR` 環境変数で上書きする。

### 問題3: ~~ダミー3関数の差し替えコードが未実装~~ → ✅ 実装済み

controller.py の `detect_person / detect_coin / detect_vegetables` はAI実装に
差し替え済み（カメラ取得 → `/predict` → 形式変換まで実装、モックでの動作確認済み）。

- サーバーURLは `app/config.py` の `PREDICT_SERVER_URL` で設定する
  （**当日はGPUサーバーの実IPに書き換えること**）
- 信頼度しきい値も config.py で調整可能（PERSON/COIN/VEGETABLE_CONF_THRESHOLD）
- コインは「前回より増えた枚数」だけを新規投入として記録する
  （置きっぱなしの硬貨を毎周期重複カウントしない）
- `python controller.py --dummy` でキーボード入力の旧ダミーモードに切り替え可能
  （サーバー・カメラなしで制御フローだけ試す用）

### 問題4: ~~録画ファイルが空になる~~ → ✅ 修正済み

セッション中のループで監視カメラのフレームを `recorder.write()` に書き込むよう
実装済み。同じフレームを人検知にも使い回すため、カメラ読み出しは1周につき1回。
monitor.mp4 に実際の監視映像が記録される（AIモード時のみ。--dummy時は空のまま）。

---

**→ 確定問題4件はすべて修正済み。当日必要な設定は `PREDICT_SERVER_URL` の
書き換えのみ。修正内容の詳細は `docs/integration_fixes.md` を参照。**

## 2. テストの段階と成功条件

一気に全部つなぐと問題の切り分けができないため、**4段階に分けて実施**する。
各段階の成功条件を満たしてから次に進む。

### 段階1: AI推論サーバー単体（担当: AI）

```bash
# サーバー起動（必ずtmux内で）
tmux new -s server
python app/web_server.py

# 別ターミナルから
curl http://localhost:8080/status
python app/simulate_raspi.py --image unattended_sales_place_images/selfsellingstation.jpg
```

**成功条件**:
- [ ] `/status` が200で応答する
- [ ] `/predict` が検出結果JSONを返す
- [ ] カメラの前に**人が立って** `person` が検出される（未実証項目）
- [ ] 実物の硬貨をカメラに見せて `100yen` 等が検出される

### 段階2: 制御フロー（ダミーのまま全体を回す・実機の重量センサーは本物）

AI差し替え**前に**、ダミーのままセッション1周を回して制御側が正常なことを確認する。
ラズパイ実機では `raspberry_pi.get_weights()` が自動で実センサー計測になるため、
**この段階で重量センサーの結合確認も同時にできる**。

```bash
# 重量センサー単体の事前確認（実測値が表示されるか）
cd app && python raspberry_pi.py

# 制御フロー1周（※必ず app/ から起動する。問題1参照）
cd app && python controller.py   # キーボードで人あり→コイン投入→人なし を模擬
```

**成功条件**:
- [ ] `python raspberry_pi.py` で実測の重量が表示される（ダミー値の警告が出ないこと）
- [ ] `sessions/<セッション名>/` が作成される
- [ ] `coin.csv` / `vegetable.csv` / `weight.csv` / `session.json` が生成される
- [ ] weight.csv の before/after に**実センサーの値**が記録されている
- [ ] `theft_checker.py` が起動し、session.json に判定結果が追記される

### 段階3: AIモードでの制御フロー（結合の本丸）

ダミー3関数のAI差し替えは**実装済み**。当日やることは設定と起動のみ：

```bash
# 1. GPUサーバーのIPを設定（app/config.py）
#    PREDICT_SERVER_URL = "http://<GPUサーバーのIP>:8080"

# 2. GPUサーバー側で推論サーバーを起動（tmux内）
tmux new -s server
python app/web_server.py

# 3. ラズパイ側でAIモード起動（※必ず app/ から）
cd app && python controller.py
```

**成功条件**:
- [ ] 人がカメラに映るとセッションが自動開始される
- [ ] 硬貨を置くと `coin.csv` に正しい金額が記録される
- [ ] 野菜の増減が `vegetable.csv` に記録される
- [ ] 人が3秒いなくなるとセッションが自動終了する

### 段階4: エンドツーエンド（Flask・LINE通知まで）

```bash
# Flask管理画面を起動
cd Flask && python app.py
```

購入シナリオと万引きシナリオを1回ずつ通す：

**シナリオA（正常購入）**: 人が来る → 野菜を取る → 正しい金額の硬貨を入れる → 立ち去る

**シナリオB（万引き疑い）**: 人が来る → 野菜を取る → 硬貨を入れない（または不足額）→ 立ち去る

**成功条件**:
- [ ] シナリオAで: 在庫が減り、売上履歴が追加され、LINE購入通知が届く
- [ ] シナリオBで: 通知履歴に万引き疑いが記録され、LINE万引き通知が届く
- [ ] Flask管理画面に在庫・売上・通知・AI認識履歴が表示される

## 3. 結合テスト全体の成功の定義

**最低ライン（これができれば結合テストは成功と言える）**:
段階3まで完了。つまり「人検知 → セッション → AI認識でCSV記録 → 万引き判定」が
人手の入力なしで1周回ること。

**理想ライン**:
段階4まで完了。LINE通知と管理画面表示まで含めた全機能のエンドツーエンド動作。

**成功とは言えない状態**:
- ダミー関数のままで「動いた」としてしまう（それは段階2であり結合ではない）
- 検出精度が低くても、フローが正しく回っていれば結合テストとしては成功。
  精度改善は別タスク（実環境画像の追加学習）として切り分ける

## 4. 起きると考えられる問題と対策

### 通信系

| 問題 | 原因 | 対策 |
|------|------|------|
| `/status` すら繋がらない | 別ネットワーク・ファイアウォール | 同じWi-Fi/LANか確認。`sudo ufw allow 8080` で開放 |
| `Connection refused` | サーバー未起動・ポート違い | サーバーのターミナルで起動ログを確認 |
| 初回リクエストだけタイムアウト | モデルロード直後で遅い | 2回目以降で判断。テスト開始前にウォームアップ1回 |
| DevContainer内のサーバーに外から繋がらない | ポートフォワーディング未設定 | ホスト側で直接 `python app/web_server.py` を実行するのが確実 |
| FlaskとAIサーバーのポート衝突 | 両方8080を使おうとした | AIサーバー=8080、Flask=5000（デフォルト）を確認。変更時は担当間で共有 |

### AI・推論系

| 問題 | 原因 | 対策 |
|------|------|------|
| 誤検出・検出漏れが多い | 実環境と学習データの乖離 | フローが回るなら結合としては合格。誤検出画像を保存しておき再学習の材料にする |
| 硬貨の種類を間違える | 照明・距離 | コインカメラを硬貨に近づける・真上から。100円玉が最も精度が高い（実証済み） |
| 人間が検出されない | 全身が写っていない | カメラから離れて上半身以上が写るように |
| `person` の class_id=-1 で他モジュールが混乱 | ID前提の実装 | 他担当には「class_nameで判定する」ことを伝える |
| 紙幣・海外硬貨クラスが誤反応 | 56クラスに紙幣等も含む | `detect_coin()` の変換時に硬貨6種以外を無視する実装にする（上のコード例参照） |

### 制御・データ連携系

| 問題 | 原因 | 対策 |
|------|------|------|
| セッションが終了しない | 人検知が誤検出し続ける | `PERSON_DISAPPEAR_TIME`（3秒）を伸ばす / 人検知のconf閾値を上げる |
| コイン認識が0.2秒周期に間に合わない | `COIN_DETECT_INTERVAL=0.2` は推論+通信には短い | interval を 0.5〜1.0 秒に緩める（`app/config.py`） |
| theft_checker がエラー | CSVの列名・形式の不一致 | 段階2でダミーが生成したCSVの形式を正とし、AI実装も同じ形式で出力する |
| Flaskが取り込まない | sessions/ のパス不一致 | `SESSIONS_DIR` 環境変数と `config.py` の `SESSION_DIR` が同じ場所を指すか確認 |
| LINE通知が来ない | トークン未設定 | `line_notify.py` の設定を確認。通知以外が動けば段階4はほぼ合格とする |

### 運用系

| 問題 | 原因 | 対策 |
|------|------|------|
| SSH切断でサーバーが止まる | フォアグラウンド実行 | 必ず `tmux` 内で起動する |
| カメラが3台ない | 機材不足 | `config.py` の index をすべて0にして1台で代用 |
| ラズパイでコードが動かない | Python 3.10未満 | `Dockerfile.rpi` はPython 3.10に修正済み。直接実行の場合は `python3 --version` を確認 |

### 想定外の事態に備えて（フォールバック）

- **ラズパイが動かない** → PCの `simulate_raspi.py --camera 0` で代行
- **カメラが動かない** → `simulate_raspi.py --image` で静止画テストに切り替え
- **ネットワークが使えない** → 全プロセスを1台のPCで起動し `localhost` で完結させる
- **controller.py の差し替えが当日間に合わない** → 段階1+段階2を別々に成功させ、
  差し替えコードはレビューだけ済ませて後日結合（段階3を持ち越し）
