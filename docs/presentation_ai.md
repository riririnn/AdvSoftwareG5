# AI認識モジュール — プレゼン資料

担当者向けの発表用まとめ。設計思想・現状・今後の方針を記載。

---

## 1. 課題設定

無人販売所では、どの商品が何個取られたか・いくら支払われたかをリアルタイムで
把握する必要がある。そのためにカメラ映像からAIで商品・硬貨・人物を自動検出する
仕組みを構築した。

---

## 2. システム構成

```
ラズパイ（カメラ撮影 → JPEG送信）
    ↓  POST /predict
Ubuntu サーバー（GPU推論 / YOLOv8）
    ↓  JSON レスポンス
他モジュール（重量センサー・売上管理・通知）
```

### 設計方針

- **ラズパイは薄いクライアント**に徹する（撮影・送信のみ）
- **重いAI推論はGPUサーバー側**で処理（Ubuntu + RTX 5060 Ti）
- 結果はJSONで返すため、他担当のモジュールと疎結合に連携できる

---

## 3. 使用技術

| 項目 | 内容 |
|------|------|
| AIフレームワーク | YOLOv8（Ultralytics） |
| 学習環境 | Ubuntu + RTX 5060 Ti + CUDA 12.8 |
| データセット | Roboflow `unattended_sales_place` v4（56クラス、13万枚） |
| 通信 | HTTP（ラズパイ → サーバー）、JSONレスポンス |
| クライアント模擬 | `app/simulate_raspi.py`（PC上でラズパイを模擬） |

---

## 4. 検出対象クラス

| カテゴリ | クラス | データソース |
|----------|----------|-------------|
| 硬貨 | 1・5・10・50・100・500円 | Roboflow統合データセット |
| 紙幣 | 1000・5000・10000円 | 同上（将来の拡張用） |
| 野菜・果物 | tomato, onion, carrot など約45クラス | 同上 |
| 人間 | person | YOLOv8事前学習済みモデル（COCO）を流用 |

### モデル戦略（2モデル並列）

| 役割 | モデル |
|------|--------|
| 野菜・硬貨・紙幣の同時検出 | 自前学習モデル（yolov8mベース） |
| 人間検出 | YOLOv8公式事前学習済み（`yolov8s.pt`、学習不要） |

野菜と硬貨は同一モデルで学習しているため、1回の推論で同時検出できる。
人間はCOCO事前学習済みモデルをそのまま使い、サーバー側で結果をマージして
1つのJSONで返す。

---

## 5. 学習結果（完了済み）

100エポック学習を完走（途中GPUドライバ障害からチェックポイント再開で復旧）。

| 指標 | 値 |
|------|-----|
| mAP50 | **82.2%** |
| mAP50-95 | 54.1% |
| Precision / Recall | 99.3% / 99.3% |

### 実測テスト結果

- 硬貨: テスト画像で**100円玉5枚を正解ラベル通りに検出**（信頼度65〜86%）
- 野菜: onion等を検出（信頼度30〜85%）
- `/predict` エンドポイント経由のエンドツーエンド動作を確認済み

## 7. 今後の方針

1. **結合テスト**でラズパイ実機・重量センサー・Web UIと接続確認
2. 実環境（販売台・実照明）での精度を測定し、必要なら実環境画像を追加学習
3. 万引き判定ロジック（AI検出結果 × 重量センサー）を重量担当と連携して実装
4. 低確度検出への対策（confidence閾値調整 or yolov8l再学習）

## 8. 想定質問と回答

**Q. なぜラズパイで推論しないのか？**
> ラズパイのCPUではYOLOの推論が遅すぎてリアルタイムに動作しないため、GPUサーバーで処理する設計にした。

**Q. 精度はどのくらいか？**
> 検証データでmAP50 = 82.2%。硬貨はテスト画像で枚数・種類とも正解と一致した。実環境での精度は結合テストで測定する。

**Q. データはどこから入手したか？**
> Roboflowの公開データセット（野菜・硬貨）を統合し、56クラス・約13万枚で学習した。

**Q. リアルタイム性は保てるか？**
> 3〜5FPSで運用予定。RTX 5060 Tiでの推論は1枚あたり数十msなので十分間に合う。

**Q. 人間の検出はどうやっているか？**
> COCOで事前学習されたYOLOv8をそのまま使用（追加学習不要）。自前モデルと並列で推論し、サーバー側で結果を1つのJSONにマージしている。

**Q. 他のモジュールとどう連携するか？**
> 推論結果をJSON形式で返すため、重量センサーや売上管理モジュールとはJSONを介して疎結合に連携できる。
